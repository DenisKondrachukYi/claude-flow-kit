# Structure

Де що лежить. Коли з'являється новий патерн — оновити тут.

## Дерево верхнього рівня

```
<project>/
├── src/              ← основний код
│   ├── features/     ← фічі, згруповані по модулях
│   ├── shared/       ← переносимі утиліти, типи
│   └── entry/        ← точка входу
├── tests/            ← тести (або co-located з кодом)
├── docs/             ← документація (цей проект)
└── <інше>
```

## Конвенції

### Іменування

- Файли: <kebab-case / PascalCase / snake_case>
- Компоненти: <>
- Хуки: <prefixу use-*>
- Типи: <>

### Організація модулів

- Один модуль = одна папка у `features/`
- Експорт через `index.ts` / `mod.go` / `__init__.py`
- Внутрішні файли не імпортувати напряму з інших модулів

### Тести

- Розташування: <co-located / окрема папка>
- Naming: <*.test.ts / *_test.go / test_*.py>
- Що тестуємо: <unit / integration / e2e>

## Границі

- `src/shared/` НЕ залежить від `src/features/`
- Генерований код у `<папка>` НЕ редагувати вручну
- Секрети НЕ комітити, використовувати `.env`

## Як додавати нове

- Нова фіча: `src/features/<name>/`
- Новий спільний утиліт: `src/shared/<category>/`
- Нова залежність: обговорити в proposal перед додаванням
