# Decisions

Лог архітектурних і продуктових рішень. Формат ADR-lite: дата, рішення, причина, альтернативи.

Додавати запис, коли:
- Вибрали бібліотеку / фреймворк / підхід
- Свідомо відмовились від чогось
- Прийняли trade-off, який неочевидний
- Змінили попереднє рішення

Формат запису (новіші зверху):

---

## YYYY-MM-DD — <Коротка назва рішення>

**Контекст:** <чому виникло питання>

**Рішення:** <що робимо>

**Причина:** <основний аргумент>

**Альтернативи, які розглянули:**
- <варіант A> — <чому ні>
- <варіант B> — <чому ні>

**Наслідки:** <що це змінює в коді, процесах, ризиках>
